# MLOps End-to-End Project: Flask · Jenkins · GitHub · AWS ECS Fargate

A complete MLOps pipeline that trains a scikit-learn model, packages it into a Docker image, runs a CI/CD pipeline in Jenkins, and deploys to AWS ECS Fargate behind an Application Load Balancer — all provisioned with Terraform.

---

## Architecture

```
GitHub Push
    → Jenkins CI/CD Pipeline
        → Lint (pylint, flake8, black)
        → Test (pytest)
        → Train Model (uv run train.py)
        → Trivy FS Scan
        → Build Docker Image (linux/amd64 via buildx)
        → Trivy Image Scan
        → Push to DockerHub
        → Deploy to AWS ECS Fargate
```

**AWS Infrastructure (Terraform):**
- VPC with public & private subnets across 2 AZs (eu-central-1)
- Application Load Balancer (public subnets)
- ECS Fargate cluster + service (private subnets)
- NAT Gateway for outbound traffic
- CloudWatch Logs
- Auto Scaling (CPU target: 70%)

## Quick Start

### 1. Train the model locally

```bash
uv init
uv add scikit-learn
uv run train.py
# Output: model/iris_model.pkl
```

### 2. Run Jenkins in Docker

```bash
cd ./custom_jenkins
docker build -t jenkins-dind .

docker run -d --name jenkins-dind \
  --privileged \
  -p 8080:8080 -p 50000:50000 \
  -v /var/run/docker.sock:/var/run/docker.sock \
  -v jenkins_home:/var/jenkins_home \
  jenkins-dind
```

Open [http://localhost:8080](http://localhost:8080) and complete the setup wizard.

### 3. Configure Jenkins

**Install plugins:** Docker, Docker Pipeline, AWS Credentials, Amazon Web Services SDK

**Add credentials:**

| ID | Kind | Value |
|----|------|-------|
| `mlops-github-token` | Username & Password | GitHub personal access token |
| `mlops-jenkins-dockerhub-token` | Username & Password | DockerHub personal access token |
| `mlops-awscred` | AWS Credentials | AWS Access Key ID + Secret |

**Install AWS CLI inside the Jenkins container (Apple Silicon):**
```bash
docker exec -it -u root jenkins-dind bash
curl "https://awscli.amazonaws.com/awscli-exe-linux-aarch64.zip" -o "awscliv2.zip" \
  && unzip -o awscliv2.zip \
  && sudo ./aws/install --update
```

**Fix Docker socket permissions:**
```bash
docker exec -it -u root jenkins-dind bash
chmod 666 /var/run/docker.sock
usermod -aG docker jenkins
exit
docker restart jenkins-dind
```

### 4. Provision AWS infrastructure

```bash
cd terraform
terraform init
terraform plan
terraform apply
```

Expected outputs:
```
alb_dns_name     = "mlops-flask-prod-alb-xxx.eu-central-1.elb.amazonaws.com"
ecs_cluster_name = "mlops-flask-prod-cluster"
ecs_service_name = "mlops-flask-prod-service"
```

### 5. Run the pipeline

Trigger a build in Jenkins — the pipeline will lint, test, build, scan, push, and deploy automatically.

Verify the deployment:
```bash
curl http://<alb_dns_name>
```

## Jenkinsfile Overview

```groovy
pipeline {
    agent any
    environment {
        DOCKERHUB_CREDENTIAL_ID = 'mlops-jenkins-dockerhub-token'
        DOCKERHUB_REGISTRY      = 'https://registry.hub.docker.com'
        DOCKERHUB_REPOSITORY    = 'dariusztrawicki/mlops-proj-01'
    }
    stages {
        stage('Clone Repository')      { ... }
        stage('Lint Code')             { ... } // pylint, flake8, black
        stage('Test Code')             { ... } // pytest
        stage('Train Model')           { ... } // uv run train.py
        stage('Trivy FS Scan')         { ... }
        stage('Build Docker Image')    { ... } // buildx --platform linux/amd64 --load
        stage('Trivy Image Scan')      { ... }
        stage('Push Docker Image')     { ... } // buildx --platform linux/amd64 --push
        stage('Deploy')                { ... } // aws ecs update-service
    }
}
```
